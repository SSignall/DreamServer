### The Code Pattern
```javascript
let result = (a.length > MAX_LEN || b.length > MAX_LEN) ? 1 : 0;

// Constant-time comparison loop (always runs MAX_LEN iterations)
for (let i = 0; i < MAX_LEN; i++) {
    // ... comparison logic that runs regardless of result
}

return result === 0;
```

### Why It's Actually Secure
- The ternary **initializes** `result` — it doesn't **return** early
- The constant-time loop still runs all `MAX_LEN` iterations regardless of the initialization value
- No early return means no timing side-channel from the bounds check
- This is the **gold standard** pattern: fold the bounds check into result, run constant-time work, return at end

## Lesson Learned
Static analysis tools (including LLM reviewers) can flag initialization ternaries as timing leaks. The key distinction:
- ❌ **Vulnerable:** `if (len > MAX) return 1;` (early return leaks timing)
- ✅ **Safe:** `let result = (len > MAX) ? 1 : 0;` (initialization, loop still runs)

When in doubt, verify the constant-time loop runs unconditionally.
