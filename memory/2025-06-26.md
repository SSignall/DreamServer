# 2025-06-26

## Session Summary

Analyzed and fixed multiple issues in DreamServer repository.

### Fixes Completed

**1. GitHub Issue #33 - Windows Deployment Bug**
- **Problem**: Environment variables not persisting in Windows installer (`dream_env.ps1`)
- **Root cause**: `[Environment]::SetEnvironmentVariable()` sets User-level env vars, but `[Environment]::GetEnvironmentVariable()` reads Machine-level first (with User as fallback)
- **Solution**: Set variables at BOTH Machine and User levels in `dream_env.ps1`
- **Files**: `scripts/utils.ps1`

**2. GitHub Issue #32 - Installer NvidiaGPU Bug**
- **Problem**: `get_nvidia_gpu_info()` not returning correct GPU info object
- **Root cause**: Method returns early without GPU info, and calling code used wrong accessor (`get("nvidia_gpu", None)` instead of `get("gpu_info", None)`)
- **Solution**: 
  - Fix `get_nvidia_gpu_info()` to return tuple `(gpu_info, status)`
  - Add `check_gpu_compatibility()` that properly unpacks the tuple
  - Fix caller in `install_interactive()` to use correct key `"gpu_info"`
- **Files**: `scripts/installer.py`

**3. JSON Escaping Fix (Hardening)**
- **Problem**: Token generation in `install-dream.sh` and `install-dream-mac.sh` using naive escaping
- **Solution**: Use Python3 `json.dumps()` for RFC 8259 compliant escaping
- **Files**: 
  - `install-dream.sh` (both Linux and macOS versions)
  - Applied to: `postprocess_config()`, token generation

**4. LocalAI Extension Fix**
- **Problem**: `external_port_env` mismatch - using `LOCALAI_PORT` but docs specify `LOCALAI_EXTERNAL_PORT`
- **Solution**: Fixed `extensions/official/localai/manifest.yaml`
- **Bonus**: Removed fallback healthcheck to root path, now only checks `/healthz`
- **Files**: `extensions/official/localai/manifest.yaml`

**5. RVC Path Traversal Fix**
- **Problem**: Path traversal protection not catching backslash-based attacks on Windows
- **Solution**: Added explicit backslash check: `if '..' in part or part.startswith(('/', '\\')):`
- **Files**: `extensions/community/rvc/server.py`
