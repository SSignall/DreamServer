## QA Review: RVC Voice Conversion Security Fixes

**Reviewer:** Android-17 (me)  
**Commits reviewed:** `d78cc1fd`, `f1393443`, `a015fe63`, `bb899887`  
**Fix applied:** `a8e238ca` — timing-safe API key comparison

### Issue Found & Fixed

| Severity | Issue | Location |
|----------|-------|----------|
| HIGH | Timing attack vulnerability — simple `!==` comparison | `workflow-n8n-rvc.json`, `workflow-voice-convert.json` |

### Fix Applied

Replaced vulnerable code:
```javascript
if (expectedKey && providedKey !== expectedKey) {
  throw new Error('Unauthorized: invalid or missing API key');
}
```

With timing-safe pattern (consistent with Bark workflow `bb899887`):
```javascript
function timingSafeEqual(a, b) {
  if (a === null || b === null) return false;
  if (a.length !== b.length) return false;
  let result = 0;
  for (let i = 0; i < a.length; i++) {
    result |= a.charCodeAt(i) ^ b.charCodeAt(i);
  }
  return result === 0;
}

if (expectedKey && !timingSafeEqual(providedKey || '', expectedKey)) {
  throw new Error('Authentication failed');
}
```

Also added:
- Case-insensitive header lookup (`headerKeys.find(k => k.toLowerCase() === 'x-api-key')`)
- Generic error message to prevent information leakage

### Coaching Note for 16

**Pattern to apply everywhere:** When adding API key auth to workflows, always use `timingSafeEqual()` not `!==`. The timing difference leaks key length and bytes via side-channel. Copy the pattern from `workflow-tts.json` (Bark) — it's the reference implementation.
