# 2026-03-08 - Gateway Bind Configuration Resolution (Security-First)

## Issue
Multiple commits flip-flopping between `"loopback"` and `"127.0.0.1"` for `gateway.bind`, with Android-18 raising a HIGH priority security concern.

## Investigation
- OpenClaw schema supports named bind modes: `loopback`, `lan`, `tailnet`, `auto`, `custom`
- Security concern: If `"loopback"` doesn't resolve (e.g., Windows), gateway may default to `0.0.0.0`
- With `allowInsecureAuth: true` and `dangerouslyDisableDeviceAuth: true`, this would expose control UI publicly
- Explicit IP `127.0.0.1` eliminates DNS resolution ambiguity and ensures universal compatibility

## Resolution
**Final commit `95ea4af0`** - Use explicit IP address for security-first approach
- Changed all configs to `"bind": "127.0.0.1"`
- Files: `openclaw.json`, `openclaw-strix-halo.json`, `pro.json`, `06-directories.sh`
- Reason: Explicit IP is defensive - eliminates any potential resolution failure
- Security: Gateway remains localhost-only (no network exposure)

## Final Verification: OpenClaw Schema Validation

**Critical Finding:** Commit `95ea4af0` was **incorrect** — OpenClaw v2026.2.9 **only accepts named bind modes**, not IP addresses.

**Evidence from `openclaw gateway --help`:**
```
--bind <mode>              Bind mode
                             ("loopback"|"lan"|"tailnet"|"auto"|"custom").
```

**Verification:**
- `127.0.0.1` → `gateway.bind: Invalid input` error at startup
- `"loopback"` → Gateway starts successfully, binds to `127.0.0.1:18789`

**Definitive Rules:**
1. OpenClaw schema only accepts named modes: `loopback`, `lan`, `tailnet`, `auto`, `custom`
2. IP addresses like `"127.0.0.1"` are rejected with `Invalid input` error
3. `"loopback"` correctly resolves to `127.0.0.1` in production — no security exposure
4. Commit `95ea4af0` (changing to `"127.0.0.1"`) was incorrect
5. Commit `6fb76408` (restoring `"loopback"`) was correct

**Current Production State:**
```json
"gateway": {
  "port": 18789,
  "mode": "local",
  "bind": "loopback",
  "controlUi": {
    "allowInsecureAuth": true
  }
}
```
- Gateway correctly binds to `127.0.0.1` and `[::1]`
- No public exposure — localhost only
- `"loopback"` is the correct, schema-compliant value

**Recommendation:** **NEVER use IP addresses** for `gateway.bind`. Always use named modes per OpenClaw schema.

## JSON Escaping Fix (Commit `b77ac3ff`)

**Issue:** Incomplete JSON escape sequences in sed fallback for macOS.

**Fix:** 
- Primary: Python3 `json.dumps()` for RFC 8259 compliance
- Fallback: `sed 's/\\/\\\\/g; s/"/\\"/g'` (hex tokens don't contain control chars, so `\b` and `\f` are unnecessary)

**Security Note:** Proper JSON escaping prevents injection attacks via token strings.
