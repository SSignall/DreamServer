# 2026-03-08 — Pre-Compaction Memory Flush (Evening)

## Security Fixes — Authentication Bypass Vulnerability

### Timeline

**Commit `f13d0010` (Original attempt):**
- Fix: `if (providedKey && !timingSafeEqual(providedKey, expectedKey || ''))`
- Issue: Only validates when client provides a key; skips check if client omits `X-API-Key` header
- Result: **Authentication bypass vulnerability** when server has `API_KEY` configured but client doesn't send header

**Commit `d370c497` (Correct fix):**
- Fix: `if (expectedKey ? !timingSafeEqual(providedKey || '', expectedKey) : providedKey)`
- Logic: Always validate when server has key configured; reject if client provides key when server has none

### Authentication Matrix

| Server Config | Client Header | `f13d0010` Result | `d370c497` Result |
|--------------|---------------|-------------------|-------------------|
| `API_KEY=secret` | `X-API-Key: secret` | ✅ Pass | ✅ Pass |
| `API_KEY=secret` | (no header) | ❌ **PASS** (BYPASS) | ❌ Fail ✅ |
| (no key) | (no header) | ✅ Pass | ✅ Pass |
| (no key) | `X-API-Key: anything` | ✅ Pass | ❌ Fail ✅ |

### Security Pattern (Final)

```javascript
// Always validate if server has key configured; reject if client provides key when server has none
if (expectedKey ? !timingSafeEqual(providedKey || '', expectedKey) : providedKey) {
  return [{ json: { valid: false, error: 'Authentication failed' } }];
}
```
