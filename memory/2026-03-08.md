## Dual GPU Detection Bug Fix (#55)

**Time:** ~00:10-00:25 EST  
**Issue:** GitHub #55 — Install routine does not auto-detect dual GPU setup  
**User Report:** 3090 + 4090 system, `nvidia-smi` shows both GPUs, but hardware scan only shows one.

### Root Cause Analysis
- `detect-hardware.sh` correctly outputs `"count": 2` in JSON for dual GPU systems
- **BUG:** `scripts/build-capability-profile.sh` line 88 hardcoded `gpu_count = 1` regardless of actual detection
- This caused the capability profile to always report single GPU, which then propagated to the installer's hardware scan display

### Fix Applied
**File:** `scripts/build-capability-profile.sh`  
**Change:** Line 88
```python
# Before (broken):
gpu_count = 1 if gpu_type not in {"none", ""} else 0

# After (fixed):
gpu_count = int(gpu.get("count", 1)) if gpu_type not in {"none", ""} else 0
```

### Verification
- Python test script confirmed `gpu_count = 2` for dual GPU case
- Commit `97bbd7c5` pushed to `dev/main`

### Post-Commit Fix (Commit `699a4126`)
**Review Finding:** Unhandled `ValueError`/`TypeError` if `gpu.count` is non-numeric or null  
**Fix:** Replaced `isdigit()` check with try/except pattern:
```python
# Before:
gpu_count_val = gpu.get("count")
gpu_count = int(gpu_count_val) if gpu_count_val is not None and str(gpu_count_val).isdigit() else (0 if gpu_type in {"none", ""} else 1)

# After:
gpu_count_val = gpu.get("count")
try:
    gpu_count = int(gpu_count_val) if gpu_count_val is not None else (0 if gpu_type in {"none", ""} else 1)
except (ValueError, TypeError):
    gpu_count = 0 if gpu_type in {"none", ""} else 1
```
**Status:** Pushed to `dev/main`

### Impact
- Hardware scan now correctly shows both GPUs (e.g., "NVIDIA GeForce RTX 3090 + NVIDIA GeForce RTX 4090 (48GB)")
- Tier detection (T4 for dual 24GB GPUs) now works correctly
- Assigned to Youness for hardware verification
