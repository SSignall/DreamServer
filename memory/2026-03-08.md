```javascript
function timingSafeEqual(a, b) {

    result |= charA ^ charB;
  }
  result |= (a.length === b.length) ? 0 : 1;
  return result === 0;
}
```

## Timing Side-Channel Fix — Commit `101ace90`

### Issue (Flagged by Android-17)
The ternary pattern `(a.length > MAX_LEN || b.length > MAX_LEN) ? 1 : 0` was incorrectly flagged as vulnerable to timing leaks. Android-17 clarified:

> **Why:** Ternaries compile to conditional branches. CPUs have branch prediction that creates measurable timing differences. XOR is arithmetic — always takes same time.

### Resolution (Commit `101ace90`)
Replaced ternary with XOR-based length check in 3 files:
- `extensions/services/n8n/workflow-n8n-ollama.json`
- `extensions/services/n8n/workflow-n8n-rvc.json`
- `extensions/services/rvc/workflow-voice-convert.json`

**Gold Standard Pattern:**
```javascript
// Constant-time length check - fold into result without early exit
const lenDiff = a.length ^ b.length;  // 0 if lengths equal, non-zero otherwise
let result = lenDiff;  // Start with length difference already folded in

// Constant-time comparison: always iterate MAX_LEN, use 0 for out-of-bounds
for (let i = 0; i < MAX_LEN; i++) {
  const charA = i < a.length ? a.charCodeAt(i) : 0;
  const charB = i < b.length ? b.charCodeAt(i) : 0;
  result |= charA ^ charB;
}

return result === 0;
```

**Why XOR is constant-time:**
- CPUs have branch prediction for conditionals
- Ternaries compile to conditional branches with timing variations
- XOR is arithmetic — always takes same CPU cycles

## Android-17 Coaching Notes

**When implementing constant-time checks in security code:**

1. **Avoid early returns** — they leak timing via branch prediction
2. **Use XOR to fold length checks** — always take same number of operations
3. **Always iterate fixed length** — don't exit early on mismatch
4. **Fold all checks into result** — final comparison `result === 0` is constant-time

**Pattern to follow:**
```javascript
const lenDiff = a.length ^ b.length;  // XOR folds length check
let result = lenDiff;
for (let i = 0; i < MAX_LEN; i++) {
  result |= (a[i] || 0) ^ (b[i] || 0);  // always MAX_LEN iterations
}
return result === 0;  // constant-time final check
```
