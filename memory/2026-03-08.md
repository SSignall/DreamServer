## JSON Escaping Clarification (23:05 EST)

**Issue (from queue):** sed fallback in `06-directories.sh` and `env-generator.sh` had logic errors:
- `\b` in sed matches word boundaries (zero-width), not backspace (ASCII 0x08)
- `\f` is non-standard in POSIX/BSD sed and may be treated as literal `f`

**Fix applied (8293cf4e):**
```bash
# BSD sed with $'...' for literal control chars
token_json=$(printf '%s' "$token" | sed 's/\\/\\\\/g; s/"/\\"/g; s/'$'\b''/\\b/g; s/'$'\f''/\\f/g; s/\t/\\t/g; s/\n/\\n/g; s/\r/\\r/g')
```

**Verification (K2.5/Todd):**
- Python3 `json.dumps()` is primary escape path (correct)
- sed fallback is defense-in-depth for hex-only tokens
- Validation order is correct (empty check before usage)
- JSON injection risk is mitigated by proper escaping before interpolation
- Token value `${token_json}` is already JSON-stringified before embedding

**OpenClaw bind value clarification:**
- `"loopback"` is a named bind mode that OpenClaw resolves internally to `127.0.0.1`
- Dream Server configs use `"127.0.0.1"` (IP address) for consistency with reference configs
- `allowInsecureAuth: true` with `127.0.0.1` binding is acceptable for local-only development

## JSON Escaping Fix (23:22 EST) - Post-Review Correction

**Trigger:** Android-18 K2.5 review flagged `\b` and `\f` patterns in sed

**Issue:** 
- sed `\b` matches word boundaries (zero-width), not backspace (ASCII 0x08)
- sed `\f` is non-standard in POSIX/BSD sed and may be treated as literal `f`

**Original fix (8293cf4e, e8ca19f2):**
```bash
token_json=$(printf '%s' "$token" | sed 's/\\/\\\\/g; s/"/\\"/g; s/'$'\b''/\\b/g; s/'$'\f''/\\f/g; s/\t/\\t/g; s/\n/\\n/g; s/\r/\\r/g')
```

**Root cause of the issue:** Even with `printf '\b'` to generate the literal control char, sed still interprets `\b` as a word boundary regex pattern, not as the backspace character.

**Final fix (b77ac3ff):**
Since tokens are hex-only (0-9,a-f,A-F), they don't contain any control characters. The sed fallback only needs to escape `\` and `"`:
```bash
token_json=$(printf '%s' "$token" | sed 's/\\/\\\\/g; s/"/\\"/g')
```

**Files modified:**
- `dream-server/installers/phases/06-directories.sh`
- `dream-server/installers/macos/lib/env-generator.sh`

**Status:** ✅ Committed and pushed to dev/main

**Key insight:** Defense-in-depth only matters if the defense is actually defending. For hex tokens, `\b` and `\f` escapes are unnecessary and buggy. Simpler is better.
