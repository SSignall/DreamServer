### Current timingSafeEqual Pattern (Correct)

```javascript
function timingSafeEqual(a, b) {
  if (typeof a !== 'string' || typeof b !== 'string') return false;
  const MAX_LEN = 256;
  if (a.length > MAX_LEN || b.length > MAX_LEN) return false;
  
  let result = 0;
  for (let i = 0; i < MAX_LEN; i++) {
    const charA = i < a.length ? a.charCodeAt(i) : 0;
    const charB = i < b.length ? b.charCodeAt(i) : 0;
    result |= charA ^ charB;
  }
  result |= (a.length === b.length) ? 0 : 1;
  return result === 0;
}
```

This is constant-time because:
- Loop always runs MAX_LEN iterations (256)
- Out-of-bounds chars use 0, not early exit
- Length difference incorporated into result, not early return
