### Dual GPU Detection Bug (Issue #55)
- **User Report:** RTX 3090 + 4090 setup, `nvidia-smi` sees both GPUs but installer only detects one
- **Root Cause:** Two bugs in `dream-server/scripts/detect-hardware.sh`:
  1. GPU name aggregation loop only added names to `name_parts` array for `count > 1`, but for unique GPUs (`count=1`), it output to stdout directly, leaving the array empty
  2. Comment incorrectly referenced "field 4" but the code correctly used field 2 for `name,memory.total` format
- **Fix Applied:**
  - Always add GPU names to `name_parts` array (both `count=1` and `count>1`)
  - Added empty array check before `printf` to avoid empty names
  - Now joins unique models with " + " (e.g., "RTX 3090 + RTX 4090")
