**Date:** 2026-03-08  
**Issue:** OpenClaw gateway bound to `0.0.0.0:18789` (publicly accessible) instead of localhost  
**Root Cause:** `gateway.bind` set to `"localhost"` which is not a valid bind mode; valid values are `"loopback"|"lan"|"tailnet"|"auto"|"custom"`

### Fix Applied

   - `/home/michael/.openclaw/openclaw.json` (my gateway, port 18789)
   - `/home/michael/android-16/.openclaw/openclaw.json` (Android-16's gateway, port 18791)

3. Verified binding changed to `127.0.0.1:18789` (localhost only, not publicly accessible)

### Verification
```
# Before fix:
LISTEN 0      511          0.0.0.0:18789      0.0.0.0:*    openclaw-gatewa

# After fix:
LISTEN 0      511        127.0.0.1:18789      0.0.0.0:*    openclaw-gatewa
```

### Valid Bind Modes
- `loopback` ‚Äî Bind to 127.0.0.1 only (localhost, most secure)
- `lan` ‚Äî Bind to all interfaces on LAN (less secure, for local network access)
- `tailnet` ‚Äî Bind for Tailscale access
- `auto` ‚Äî Auto-detect based on environment
- `custom` ‚Äî Use custom host configuration
